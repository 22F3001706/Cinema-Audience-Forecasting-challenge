1. Executive Summary

This report documents a complete solution for a cinema audience forecasting competition. The objective was to predict daily audience counts for movie theaters across a two-month prediction window (March to April 2024) using historical data spanning January 2023 to February 2024.

The solution involved extensive exploratory data analysis, feature engineering, and comparative modeling using SARIMA, XGBoost, and LightGBM. XGBoost emerged as the optimal model, achieving a validation Mean Absolute Error (MAE) of 14.41 and Root Mean Square Error (RMSE) of 20.66. The final submission successfully generated forecasts for all 38,062 theater-date combinations in the test set.

2. Problem Definition

2.1 Objective

Develop a forecasting model to predict daily audience counts for movie theaters for the period March 1, 2024 to April 22, 2024.

2.2 Input Data

Historical audience and booking data from two platforms (BookNow and CinePOS) covering January 1, 2023 to February 28, 2024.

2.3 Output Format

A submission file containing predictions for each theater-date combination in the format: book_theater_id_YYYY-MM-DD with associated audience count.

2.4 Evaluation Metric

The competition used Root Mean Square Error (RMSE) as the primary evaluation metric.

5. Feature Engineering

5.1 Temporal Features

  • day_of_week - Day of week (0=Monday to 6=Sunday)
  • month - Calendar month (1-12)
  • quarter - Calendar quarter (1-4)
  • day_of_year - Day of year (1-365/366)
  • week_of_year - ISO week number
  • is_weekend - Weekend indicator (1 for Saturday/Sunday)
  • is_month_start - First day of month indicator
  • is_month_end - Last day of month indicator

5.2 Cyclical Encodings

sin_day_of_year = sin(2 * pi * day_of_year / 365)
cos_day_of_year = cos(2 * pi * day_of_year / 365)

5.3 Lag Features

  • lag_1 - Audience count from previous day
  • lag_2 - Audience count from 2 days prior
  • lag_7 - Audience count from 7 days prior (same day previous week)
  • lag_14 - Audience count from 14 days prior (two weeks ago)

5.4 Rolling Statistics

  • roll_mean_7 - 7-day moving average (excluding current day)
  • roll_mean_14 - 14-day moving average (excluding current day)

5.5 Complete Feature Set

The final feature set comprised 16 variables plus theater_encoded for categorical representation.

8. Prediction Strategy

8.1 Recursive Forecasting Approach

Due to the dependency of lag features on previous predictions, a recursive forecasting strategy was implemented:

  1. For each theater independently, concatenate historical data with future prediction dates
  2. Initialize future audience_count values as NaN
  3. Train XGBoost model on complete historical data
  4. Sequentially predict each future date and update the dataframe
  5. Recompute lag and rolling features after each prediction
  6. Collect predictions for all theaters and format for submission

8.2 Handling of Negative Predictions

Predictions were capped at zero (audience_count cannot be negative) using max(prediction, 0).

10. Key Insights

10.1 Data Quality Insights

  • The booknow_theaters dataset has significant missing book_theater_id values (62%), limiting its utility for theater-level feature integration.
  • The cinePOS_theaters dataset lacks coordinates for 82% of theaters, making spatial features unreliable without extensive imputation.
  • Duplicate records exist across multiple datasets and were removed to prevent aggregation errors.
  • The mapping file covers only 150 theater pairs, providing limited cross-platform integration.

10.2 Temporal Insights

  • The audience time series is stationary, enabling direct application of standard time series models without differencing.
  • Clear weekly seasonality exists with a 7-day cycle, justifying the use of weekly lag features.
  • The training period (14 months) provides sufficient data for capturing both short-term patterns and long-term trends.

10.3 Behavioral Insights

  • The majority of bookings (70%) are made in advance of the show date, indicating planning behavior among audiences.
  • Sunday accounts for 13.32% of all bookings, making it the most popular booking day.
  • Ticket sales per transaction are typically small (median of 3 tickets), suggesting individual or small-group attendance patterns.

11. Limitations and Future Work

11.1 Current Limitations

  • Cross-Platform Integration: The limited mapping between BookNow and CinePOS theaters (150 pairs) prevents full utilization of CinePOS booking data.
  • Spatial Features: High missing rates in latitude/longitude data prevent reliable geographic analysis.
  • Computational Cost: Recursive forecasting for 827 theaters requires significant computation time.
  • Single Model Approach: No ensemble method was implemented to combine predictions from multiple models.

11.2 Future Improvements

  • Enhanced Feature Engineering: Holiday indicators, weather data integration, competitor analysis, movie release schedules.
  • Alternative Modeling Approaches: LSTM/GRU neural networks, Prophet by Facebook, ensemble methods.
  • Technical Improvements: Parallel processing for theater-wise predictions, GPU acceleration, feature importance analysis.
  • Validation Framework: Walk-forward validation, multiple validation windows, confidence interval estimation.

from statsmodels.tsa.stattools import adfuller

def adf_test(series):
    result = adfuller(series)
    print('ADF Statistics: {}'.format(result[0]))
    print('p-value: {}'.format(result[1]))
    if result[1] <= 0.05:
        print("Stationary - reject null hypothesis")
    else:
        print("Non-stationary - cannot reject null hypothesis")
